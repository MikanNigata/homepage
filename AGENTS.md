# AGENTS.md

## Read Order

1. UI変更時は必ず root の `DESIGN.md` を最初に読む。
2. `references/figma.DESIGN.md` と `references/vercel.DESIGN.md` が存在する場合は、それも確認したうえで `DESIGN.md` と整合する判断を行う。
3. 参照元と root の `DESIGN.md` が衝突する場合は、このプロジェクト用に再解釈した root の `DESIGN.md` を優先する。

## Product Priorities

- このサイトは高専内LT会の参加者募集・登壇募集のための学生向けイベントサイト。
- 学生向けイベントサイトなので、親しみやすさ > 威圧感。
- 派手さより読みやすさを優先する。
- 技術イベントらしさは必要だが、企業LPや採用サイトのように硬くしすぎない。
- 初参加者が「見に行ってみようかな」「発表してもいいかも」と思える空気を壊さない。

## Implementation Rules

- 新規セクション追加時は既存の spacing / typography / card / button パターンを再利用する。
- イベント情報はできるだけ1箇所に集約し、文言差し替えしやすく保つ。
- UI判断で迷った場合は、装飾より情報設計、独自性より参加しやすさを優先する。
- 仮文言でも、そのまま公開されても不自然でない日本語にする。
- 作業前に対象ファイルを確認し、実装後は変更点を要約する。

## Final Check

- 実装の最後に、`DESIGN.md` との整合性をセルフレビューする。
- レイアウト、色、タイポ、CTA、モバイル可読性、FAQ/カードの扱いが `DESIGN.md` の方針からズレていないか確認する。
