# DESIGN.md

## このサイトの目的

高専内LT会の参加者募集・登壇募集を行う1ページサイトをつくる。最優先は、学生が「気軽に参加できそう」「登壇してもよさそう」と感じること。情報の見通しを保ちながら、技術イベントらしい軽快さと少しの高揚感を持たせる。

## 想定ユーザー

- 高専生
- 教員
- 技術に興味のある学生
- 初参加者

## デザインソース

- 第一参照元: `references/figma.DESIGN.md`
  - 親しみやすさ
  - 軽快さ
  - やわらかい印象
  - カラーバランス
- 第二参照元: `references/vercel.DESIGN.md`
  - 情報整理
  - ヒーローの密度
  - タイポの整理
  - CTAの明快さ

参照するのは design language / spacing / typography / component patterns のみ。ブランド、ロゴ、コピー、固有表現は真似しない。完全コピーはしない。

## トーン

- 親しみやすい
- 清潔
- 少しワクワクする
- 技術イベントらしい
- ただし企業LPっぽくしすぎない

## 避けるもの

- 採用ページ感
- B2B営業LP感
- 過度な未来感
- 暗すぎる配色
- 読みにくい装飾
- 情報より演出が前に出る構成

## レイアウト原則

- 日本語の1ページサイトとして構成する。
- 1セクション1役割を徹底し、内容を詰め込みすぎない。
- Hero でイベントの要点を掴めるようにし、その下で不安を順に解消する。
- セクション順序は「期待を作る → 内容を理解する → 自分ごと化する → 行動する」の流れにする。
- デスクトップでは情報をきれいに整理し、スマホでは縦方向に自然に読めることを優先する。

## 色の原則

- ベースは明るい背景を使う。
- 本文は濃いグレー〜黒寄りで十分なコントラストを確保する。
- アクセントカラーは1〜2色に絞る。
- Figma系の軽やかさを出すため、アクセントは少し鮮やかでもよい。
- Vercel系の整理感を出すため、広い面は白や薄い面色で静かに保つ。
- 暗い背景の大面積使用は避ける。暗色は区切りや強調に限定する。

## タイポグラフィ原則

- 見出しは力強く、少しだけタイトな印象にする。
- 本文は読みやすさ最優先で、過度に小さくしない。
- 英字の小見出しやラベルはイベント感を足すために使ってよいが、主役は日本語情報。
- 情報の階層は「Hero見出し / セクション見出し / カード見出し / 本文 / 補足」で明確に分ける。
- 数字、日時、会場などの要点は視線で拾いやすくする。

## 余白ルール

- セクション間は広めに空ける。
- Hero はファーストビューで窮屈に見えない余白を取る。
- ボタン、カード、FAQ の内部余白は十分に確保する。
- 情報量が多い場合も、行間とブロック間余白で圧迫感を消す。

## ボタンルール

- CTA は主従を明確にする。
- 主CTAは「登壇する」「参加する」など行動がひと目で分かる文言にする。
- ボタン形状は角丸を強めにし、親しみやすい印象を持たせる。
- 主要CTAは視認しやすく、補助CTAは静かに整理する。
- モバイルでも押しやすい高さと余白を確保する。

## カードルール

- カードは角丸あり、軽い境界か薄いシャドウで整理する。
- カードは情報整理のために使い、装飾のためだけには増やさない。
- Hero の周辺カードは、要点整理や参加判断に直結する内容に限定する。
- FAQ は開閉しやすく、シンプルに保つ。

## Hero のルール

Hero には以下を含める。

- イベント名
- 一言で伝わる短い説明
- 開催日時
- 会場
- 参加CTA
- 登壇CTA

Hero は「イベントの空気をつくる場所」でありつつ、「参加するかどうかを判断する場所」でもある。Figma系の軽やかさとして少し華やかなアクセントを入れつつ、Vercel系の整理感で情報をきれいに並べる。

## セクション順序

1. Hero
2. 開催概要
3. LT会とは
4. こんな人におすすめ
5. 登壇募集
6. タイムテーブル
7. FAQ
8. アクセス
9. フッター

## スマホ時の優先順位

- ファーストビュー内でイベント名、日時、会場、CTA を把握できることを優先する。
- ナビゲーションよりも Hero の理解を優先する。
- カードは横並び前提にせず、1列でも読みやすい密度にする。
- 長文は短い段落に分割し、見出しで流れを補助する。
- タップ対象は十分なサイズを確保する。

## コピーライティングの方針

- 難しい言葉を避ける。
- 初参加でも怖くない印象をつくる。
- 「発表してもいいかも」と思える後押しを入れる。
- 技術イベントらしい言葉は残すが、内輪感や上級者限定感は出しすぎない。
- テンションは高すぎず、前向きでフラットにする。

## このプロジェクトでの再解釈

- Figma系の参照は、白基調の中に柔らかいアクセント色と丸みを入れる方向で取り込む。
- Vercel系の参照は、ヒーローの情報整理、強い見出し、明快なCTA、静かなサーフェス階層として取り込む。
- 最終的なUIは「やわらかい技術イベントサイト」。創作ツールの楽しさと、開発者向けLPの整理感を中間で合わせる。
